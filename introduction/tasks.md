### Блок 1: Hello, World! и основы синтаксиса (1–5)

1.  **Hello, `[ваше_имя]`:** Напишите программу, которая выводит «Hello, World!», а затем модифицируйте её, чтобы она выводила «Привет, [Ваше Имя]».
2.  **Пользовательский ввод:** Запросите у пользователя его возраст (число) и выведите: «Через 10 лет вам будет X лет».
3.  **Псевдонимы (Alias):** Объявите тип `MyString` на основе `string` и создайте переменную этого типа.
4.  **Неиспользуемые переменные:** Создайте 3 переменные, используйте только 2 из них и скомпилируйте код, убедившись в ошибке. Затем исправьте её (`_`).
5.  **Множественное присвоение:** Поменяйте местами значения двух переменных `a` и `b` без использования третьей переменной (используя `a, b = b, a`).

### Блок 2: Типы данных и константы (6–10)

6.  **Явное преобразование:** Сложите `int` (10) и `float64` (5.5), правильно приведя типы, и выведите результат.
7.  **Переполнение (uint8):** Присвойте переменной типа `uint8` значение 300 и объясните результат (или получите ошибку компиляции).
8.  **Константы и iota:** Создайте константы для дней недели (Monday=1, Tuesday=2...) с помощью `iota`.
9.  **Rune vs Byte:** Объясните разницу на примере: строка "Привет" — выведите длину (`len`) и количество рун (`utf8.RuneCountInString`).
10. **Чтение битов:** Напишите код, который проверяет, установлен ли 3-й бит в числе 13 (1101).

### Блок 3: Управляющие конструкции (11–16)

11. **If с инициализацией:** Напишите программу, которая проверяет четность числа, объявив переменную внутри условия `if`.
12. **Switch без условия:** Напишите аналог `if-else` цепочки с помощью `switch {}` для оценки баллов (90+ = "A", 80-89 = "B").
13. **Switch с типами:** Напишите функцию, которая принимает `interface{}` и через `switch type` определяет, передан ли `int`, `string` или `bool`.
14. **For как while:** Используя только `for`, выведите числа от 1 до 10, пропуская 5 (используйте `continue`).
15. **Вложенные циклы:** Выведите таблицу умножения 5x5 (1x1=1 ... 5x5=25).
16. **Goto:** Напишите код, использующий `goto` для имитации повтора ввода пароля (плохая практика, но поучительно).

### Блок 4: Массивы, слайсы и карты (17–23)

17. **Массивы фиксированные:** Создайте массив из 5 элементов. Попробуйте добавить 6-й — получите ошибку компиляции.
18. **Слайсы (make):** Создайте слайс строк с помощью `make` (длина 3, емкость 5) и добавьте 4 элемента с помощью `append`.
19. **Срез (slicing):** Дан массив `[10]int`. Создайте слайс со 2-го по 5-й элементы и измените элемент слайса — проследите, как изменился исходный массив.
20. **Копирование:** Скопируйте один слайс в другой с помощью `copy` и убедитесь, что они независимы.
21. **Удаление из слайса:** Напишите функцию для удаления элемента по индексу `i` из слайса целых чисел.
22. **Карта (map) vs struct:** Создайте `map[string]int` для хранения цен на товары. Напишите проверку существования ключа через `v, ok := m[key]`.
23. **Пустая карта:** Объясните разницу между `var m map[string]int` и `m := make(map[string]int)` (nil vs empty) — попробуйте добавить элемент в nil-карту (панка).

### Блок 5: Функции (24–30)

24. **Несколько возвратов:** Напишите функцию `Divide(a, b int) (int, error)`, которая возвращает 0 и ошибку при делении на ноль.
25. **Именованные возвраты:** Напишите функцию `getCoords() (x, y int)`, используя naked return.
26. **Функция как значение:** Присвойте функцию `func(a, b int) int { return a+b }` переменной `op` и вызовите её.
27. **Функция высшего порядка:** Напишите функцию `apply(nums []int, op func(int) int)`, которая применяет `op` к каждому элементу.
28. **Функция, возвращающая функцию:** Напишите генератор `func counter() func() int`, который при каждом вызове увеличивает счетчик на 1.
29. **Variadic (...int):** Напишите функцию `sum(nums ...int) int`, которая суммирует любое количество аргументов.
30. **Defer:** Напишите функцию, которая открывает "файл" (симулируем принтом), обрабатывает данные и закрывает его через `defer`, даже если произошла паника.

### Блок 6: Указатели (31–35)

31. **Zero value указателя:** Объявите указатель `var p *int` и проверьте его значение (nil). Попробуйте разыменовать — перехватите панику.
32. **Изменение через указатель:** Напишите функцию `func zeroVal(val int)` и `func zeroPtr(ptr *int)` — покажите, что меняет оригинал, а что нет.
33. **New:** Создайте указатель на int через `new(int)`, присвойте значение и выведите.
34. **Указатель на структуру:** Создайте `struct Point{X, Y int}` и функцию `move(p *Point, dx, dy int)`.
35. **Слайс указателей:** Создайте слайс из 3 указателей на int и присвойте каждому значение.

### Блок 7: Структуры и методы (36–40)

36. **Встраивание (Embedding):** Создайте структуру `Person` (Name, Age) и встройте её в структуру `Employee` (Salary).
37. **Методы на структуре:** Реализуйте для `Rectangle` (Width, Height) методы `Area()` (значение) и `Scale(factor float64)` (указатель). Объясните разницу.
38. **Конструктор:** Напишите каноничную функцию-конструктор `NewCar(brand string, year int) *Car`.
39. **Теги полей:** Добавьте теги `json:"name"`, `json:"age,omitempty"` к структуре и сериализуйте её в JSON (`encoding/json`).
40. **Сравнение структур:** Сравните две одинаковые структуры через `==`. Создайте структуру с полем-слайсом и попробуйте сравнить — что произойдет?

### Блок 8: Интерфейсы (41–45)

41. **Пустой интерфейс:** Напишите функцию `describe(i interface{})`, которая выводит `Type: T, Value: V`.
42. **Утверждение типа:** Извлеките `int` из `interface{}` с проверкой `value, ok := i.(int)`.
43. **Набор методов:** Определите интерфейс `Speaker` с методом `Speak()`. Реализуйте его для структур `Dog` (Гав) и `Cat` (Мяу).
44. **Встраивание интерфейсов:** Создайте интерфейс `ReadWriter`, встроив в него `Reader` и `Writer`.
45. **Nil интерфейс vs nil значение:** Объясните, почему функция, возвращающая `error`, может вернуть `*MyError(nil)`, но `err != nil` будет `true`.

### Блок 9: Обработка ошибок и паника (46–48)

46. **Оборачивание ошибок:** Напишите цепочку: `func A() error -> B() error -> C() error`, оборачивая ошибки через `fmt.Errorf("...: %w", err)`.
47. **Errors.Is:** Создайте собственную ошибку `var ErrNotFound = errors.New("not found")` и проверьте через `errors.Is`.
48. **Recover:** Напишите функцию, которая паникует с сообщением, и перехватите панику в `defer recover()`, вернув ошибку.

### Блок 10: Пакеты, горутины и простой тест (49–50)

49. **Свой пакет:** Создайте пакет `mathops` с функцией `Add`. Импортируйте его в `main` и используйте.
50. **Горутина и канал:** Запустите горутину, которая вычисляет квадрат числа и отправляет результат в канал `ch <- result`. В `main` дождитесь и выведите результат.
